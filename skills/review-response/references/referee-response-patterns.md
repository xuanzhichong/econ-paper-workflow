# Referee Response Patterns for Empirical Economics and Agricultural Economics

Use these patterns for journal revisions and R&R letters.

These are response-letter patterns for empirical economics and agricultural economics papers.  
They are not conference rebuttal templates.

The purpose of a good referee response is not only to sound professional, but to make each reply:

- clear
- traceable
- proportionate to the actual revision
- aligned with the manuscript change, if any

---

## Comment-Level Pattern

A strong per-comment response usually does the following:

1. thank the referee
2. restate the concern briefly
3. state the response stance clearly
4. answer the substance directly
5. describe the manuscript change, if any
6. cite the new section, table, appendix, or page-line location
7. state any remaining scope limit when relevant

A good response should make it easy to identify:

- what the referee is worried about
- whether the authors agree, clarify, partially accommodate, or decline
- what changed in the manuscript
- where the change can be found

---

## Core Response Stances

Use one of these stances explicitly or implicitly in each response:

- `agree and revise`
- `clarify and revise exposition`
- `clarify without revising`
- `partially accommodate`
- `respectfully decline with justification`

Do not blur these categories.  
A response becomes hard to evaluate when it is unclear whether the manuscript was actually changed.

---

## Useful Phrases

### General acknowledgement

- "We thank the referee for raising this concern."
- "We appreciate this helpful suggestion."
- "We thank the referee for pointing out this issue."

### Agree and revise

- "We agree that the original draft did not make this point sufficiently clear."
- "To address this concern, we now ..."
- "We have revised Section X to clarify ..."
- "We added Table A.X and revised Section Y to ..."

### Clarify existing logic

- "Our interpretation remains unchanged, but we agree that the original exposition was too compressed."
- "We now explain this identification logic more explicitly in ..."
- "We have revised the text to make the sample construction / scope condition / mechanism boundary clearer."

### Partial accommodation

- "We agree with the concern and have incorporated part of the requested analysis."
- "Specifically, we now provide ..."
- "We do not implement the full request because ..., and we now state this limitation directly."

### Respectful decline

- "We respectfully do not implement this request because ..."
- "Implementing this suggestion would change the estimand / weaken identification / require unavailable data."
- "We have instead clarified this boundary and added a brief discussion of the tradeoff."

### Cautious evidence language

- "The new evidence is consistent with ..."
- "These results reduce the concern that ..."
- "The evidence is suggestive rather than definitive."
- "The findings are mixed, and we now state this more clearly in the manuscript."

## Common Economics Categories

Common referee comment types include:

- identification challenge
- robustness request
- mechanism clarification
- data construction question
- sample restriction or sample construction issue
- exposition or organization issue
- literature positioning issue
- causal-language or interpretation-strength issue
- appendix, extra table, or supplementary-analysis request

These categories are useful because they often imply different response strategies.

---

## What a Good Response Should Make Clear

For each comment, a strong response should make clear:

1. **the concern**
2. **the stance**
3. **the substantive answer**
4. **the manuscript change**, if any
5. **the exact location of the change**
6. **the remaining limitation**, if the request is only partially addressed

If these elements are not visible, the response is usually too vague.

---

## Tone Rules

- be respectful
- be direct
- do not over-apologize
- do not promise changes that are not actually made
- if evidence is mixed, say so
- distinguish revision from clarification
- do not overstate what a new robustness check proves
- do not imply full agreement when only partial accommodation was made

A strong economics response is professional, specific, and non-defensive.

---

## Common Mistakes

Avoid the following:

- saying "we revised accordingly" without saying what changed
- thanking the referee but not answering the substance
- claiming agreement while making no real revision
- citing a location that does not clearly contain the described change
- using vague phrases such as "the manuscript has been improved throughout"
- overstating the contribution of a new appendix table or robustness check
- treating a response-only explanation as if it were a manuscript revision
- sounding dismissive when declining a suggestion

---

## Working Rule

A useful referee response should allow a reader to answer five questions quickly:

1. What is the referee asking?  
2. What is the author's stance?  
3. What is the substantive answer?  
4. What changed in the manuscript?  
5. Where can that change be found?  

If those five answers are not easy to identify, the response pattern is probably too weak.
