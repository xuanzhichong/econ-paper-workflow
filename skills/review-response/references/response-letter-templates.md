# Economics Response Letter Templates

These are journal-style response templates for economics and agricultural economics papers, not conference-style response templates.

Use them for `reviewer response` and `R&R` writing.  
The goal is not to sound apologetic or combative, but to make each response:

- clear
- traceable
- professional
- proportionate to the actual revision

A good response should always distinguish between:

- what was revised in the manuscript
- what was clarified in the response
- what was only partially accommodated
- what was respectfully declined

---

## Per-Comment Skeleton

    Comment [R1.C1]

    We thank the reviewer for raising this point about [issue].

    Stance:
    [Agree and revise / Clarify and revise exposition / Clarify without revising / Partially accommodate / Respectfully decline]

    Response:
    [Direct substantive answer. State the logic clearly and address the core concern rather than only acknowledging it.]

    Revision:
    [Describe the manuscript change, if any. State exactly what was added, revised, removed, or clarified.]

    Location:
    [Section / subsection / page / table / appendix reference. Use page-line references when available.]

    Appendix or robustness note:
    [Only if relevant. Mention new appendix material, robustness checks, or supplementary discussion.]

---

## Accept and Revise

Use when the reviewer identifies a real weakness in exposition, design explanation, empirical presentation, or supporting analysis, and the manuscript has been revised accordingly.

    We agree that the original draft did not make this point sufficiently clear. We have revised the manuscript to [change]. The revised text now explains [substance] more explicitly. We also added [table/check/appendix item] to address the underlying concern.

### Slightly stronger version

    We thank the reviewer for this helpful suggestion. We agree that the original manuscript did not address this issue with sufficient clarity. In response, we have revised [section] to explain [substance], and we have added [new table / robustness check / appendix discussion] to address the concern more directly.

---

## Clarify and Revise Exposition

Use when the underlying logic remains the same, but the reviewer correctly identified that the original exposition was too compressed, ambiguous, or difficult to follow.

    We appreciate this comment. Our interpretation remains unchanged, but we agree that the original exposition was too compressed. We have revised the text to explain [identification logic / sample construction / mechanism boundary] more explicitly and added a cross-reference to [section/appendix].

### More design-focused version

    We thank the reviewer for pointing out that this part of the argument was not sufficiently transparent in the original draft. Although our empirical strategy remains unchanged, we have revised the manuscript to state the identifying logic, scope condition, and supporting discussion more explicitly in [location].

---

## Clarify Without Revising the Manuscript

Use when the reviewer concern can be answered directly, but no actual manuscript-side revision is necessary or appropriate.

Important: use this only when the lack of revision is genuine and defensible.

    We appreciate this comment. Our reading of the issue is as follows: [direct clarification]. We do not revise the manuscript on this point because the current specification / definition / scope is already consistent with the paper’s estimand and design. To avoid overstating the revision, we address the point here through clarification rather than manuscript change.

### More cautious version

    We thank the reviewer for this observation. We believe the current manuscript already adopts the appropriate definition / estimand / scope for this question. We therefore do not make a substantive manuscript change on this point, but we clarify here that [direct explanation].

---

## Partially Accommodate

Use when the concern is valid, but the full requested analysis or redesign is not feasible, not identified, or not appropriate for the paper’s scope.

    We agree with the concern and have incorporated part of the requested analysis. Specifically, we now provide [new check], which addresses [core issue]. We do not implement [additional request] because [constraint], and we now state this limitation directly in the manuscript.

### Economics-specific version

    We agree that this concern is important. In response, we now add [robustness check / appendix table / clarification], which directly addresses [core issue]. We do not implement the full requested extension because doing so would change the estimand / weaken identification / require unavailable data. We now make this limitation explicit in the manuscript.

### Data-limitation version

    We thank the reviewer for this suggestion. We are unable to implement the exact requested analysis because [data limitation / missing variable / policy-timing limitation]. However, we now provide [partial accommodation], which addresses the central concern as far as the available data allow.

---

## Respectfully Decline with Justification

Use when the requested change would materially weaken the design, move the paper outside its scope, alter the estimand, or require unavailable data.

    We understand the reviewer’s suggestion, but we do not adopt it because it would change the estimand / weaken identification / require unavailable data. We have instead clarified this scope condition in the manuscript and added a short discussion of the tradeoff.

### Slightly fuller version

    We appreciate the reviewer’s suggestion and understand the motivation behind it. We nevertheless do not adopt this change because [reason]. In our judgment, implementing it would [change the estimand / undermine comparability / weaken identification / require data we do not observe]. To make this boundary more transparent, we now clarify the relevant scope condition and tradeoff in [location].

---

## Response-to-Identification Concern Template

Use when the reviewer questions causal interpretation, design credibility, or identifying assumptions.

    We thank the reviewer for raising this important concern about identification. We agree that the credibility of the design depends on [assumption / threat]. In response, we now add [event-study evidence / placebo test / alternative specification / appendix discussion], which directly addresses this issue. We also revise the manuscript to state the identifying assumption and its limits more explicitly.

---

## Response-to-Robustness Request Template

Use when the reviewer asks for additional checks tied to a specific threat.

    We appreciate this suggestion. To address the concern that [specific threat], we now report [new robustness check] in [location]. The resulting estimate remains [direction / approximate magnitude], which reduces the concern that the baseline result is driven by [specific issue].

---

## Response-to-Literature-Positioning Concern Template

Use when the reviewer argues that the paper misstates the literature, misses key references, or overclaims novelty.

    We thank the reviewer for pointing out this literature issue. We have revised the discussion in [location] to better distinguish what the existing literature already establishes from what remains unresolved. We also added the suggested references and have narrowed our contribution claim accordingly.

---

## Response-to-Data or Sample Construction Concern Template

Use when the reviewer questions sample restrictions, merging, attrition, or variable definitions.

    We appreciate this comment. In response, we now clarify the sample construction process in [location], including [restriction / merge rule / exclusion / missing-data treatment]. We also added [appendix table / flow description / note] so that the estimation sample can be traced more transparently.

---

## Tone Discipline

Use the following tone rules throughout the letter:

- respectful but not submissive
- direct but not defensive
- transparent about what changed versus what did not
- explicit when a revision was made
- explicit when only clarification was provided
- cautious when the requested change cannot be fully implemented

Do not:

- claim that a revision was made if it was not
- say “we fully agree” when the response is only partial
- overstate what a new robustness check proves
- imply that the reviewer concern is resolved if only part of it was addressed

---

## Common Mistakes to Avoid

- vague language such as “we revised accordingly” without saying what changed
- no location reference after describing a revision
- mixing manuscript revision with response-only explanation
- agreeing too broadly when only a partial accommodation was made
- claiming that the concern is resolved without traceable evidence
- using citations that do not directly answer the reviewer’s concern
- sounding defensive when declining a suggestion
- overstating concession beyond what the paper actually changed

---

## Working Rule

For each reviewer comment, the response should make clear:

1. what the concern is  
2. what your stance is  
3. what changed, if anything  
4. where the change appears  
5. what remains outside the paper’s scope, if relevant  

If those five things are not easy to identify, the response is probably not yet ready.
